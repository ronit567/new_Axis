◆ KEYSTONE

The UI Listing type was designed for a screen, not a database. It nests a seller object, stores postedAgo as "3d ago", and carries saved per-row — the DB has seller_id, created_at, and a saved_listings join. Nothing renders until a mapper layer (AX-110) bridges the two. Every screen ticket depends on it.
